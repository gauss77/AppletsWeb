Intro

java2script is a great java to javascript translator. Nevertheless, is imposible to have a perfect java to javascript translator, all we can spect are aproximations: javascript just doesn't support some  java features, like java.util.File, Threads and some features.

The Mauve Project is a collaborative effort to write a free test suite for the Java™ class libraries. It has its own testing tools and has some tests for major java2 features.

In this project, j2sMauve, we run all of mauve tests for packages in java.lang and java.util in java2script. The red rows correspond to tests interrupted by native javascript exceptions.

About the tests

a nauve test executed in java2script can finish in the following ways:
1) the test ends OK, with no exceptions. In this case all assertions where  checked.
2) the test ends with a native exception. This is a common case. Can be caused because a native nullpointer exception, anot implemented java method, etc. In this case some or none asserts were checked, but the tests was interrupted. (a red row)

In the case of fail assers (bugs), each failed assert message is printed on the column Failed asserts names (comma separated)

How to run these tests?

download j2sMauve.tar.gz  and execute for example org.sgx.j2s.mauve.report2.JavaLangReport with a java2script newer version (svn version at this moment). That class is in sgxReport source folder. Please note that at the time of this writing, you have to use a svn version of java2script.

Advanced Notes

I was playing with mauve test suite in java2script a time ago, but native javascript exceptions made impossible to make a good reporting tool. Fortunately, newer versions of java2script made possible to catch native exceptions like this:

try {
test();
} catch(Exception javaException) {
}catch(Throwable javascriptException){

This is exactly what the test report tool does.

1) Some parts of mauve test suite where deleted like, java.sql because have not porpuse in java2script.
2) java2script unsupported features where erased of mauve suite for it to work in java2script
3) some tests of classes known to be unsupported by java2script, like reflection, File, etc  are ommited in the tests results
